WHY I CHOOSE SOFTWARE DEVELOPMENT AS A CAREER

Software development is a vast field and one that is creative, demanding and highly paying so choosing it as a career gives you the opportunity to learn a lot about technology and also some about business. Here are some reasons why I choose software development as my career.

  1. Passion for software developing and computer at large

Passion is always the leading character for doing any thing, so ask yourself “ Do you have passion and love for software development?”. Well as for me , I do love software development and I have loved it ever since I was young. So love for software development is the first reason why I choose software development.

2.Test your problem- solving skills

As a software developer you constantly provide solutions for user’s problems and there is nothing more satisfying than solving a problem that has been around for a while and nobody or a few knows how to solve.

3.Collaboration across teams

As a Developer , you spend a lot of time collaborating with colleagues from multiple different departments, which means you have the chance to learn from others. In addition to technical skills , you need to be a great communicator and team player and hence I like collaborating with people.

4.Continuous learning opportunities

Technology is extremely fast moving, so developers need to be able to learn the new programming languages and technologies and also adapt to the continually changing world. This also helps developers to improve their knowledge about business and sectors that they are working in, as they are also ever-changing.

Highly good payment

Software Development in Uganda is a highly earning profession and highly demanding as well.

According to Glassdoor, 6 of the 15 highest paying jobs of 2019 and 2020 are in the software development industry.

Get creative with code

There are many ways in which Software Developers can be creative, including the way you think to solve a problem for example, you need to be able to put yourself in the users’ shoes in order to provide a solution. You need to create innovative new systems and functionalities and you get to play around and learn new technologies.

All of this is being creative as a Software Developer.

STEPS TO FOLLOW WHEN SETTING UP A GITHUB PROJECT(REPOSITORY) FOR BEGINNERS.

GitHub is a code hosting platform for version control and collaboration. It lets you and others work together on projects from anywhere.

To put your project up on GitHub, you will need to create a repository for it to live in . They are two ways of creating a repo . you can create a repo direct on git hub , secondly create a repo locally using git and then push it to git hub. It is rather best for beginners to create using git because it will them get used and learn more of the codes. So am going to take you through the steps of creating a repo using git.

Install git and create a GitHub account

The first step you have to do is to install git and create an account on GitHub. Creating an account on GitHub is for free.

Create a local git repository

When creating a new project on your local computer using git, you will first create a new repository.

First, open up git bash and move to where you want to place your project using the cd command which means change directory.

  • cd Desktop/ command is to change directory to Desktop
  • mkdir(make directory) command is to create a repo name on desktop
  • cd ( name of the repo) to change directory from desktop to repo created

Add a new file to the repo

Add a file to the repo using a “ touch command”. For our case , we are adding a Readme.md file

  • touch Readme.md
  • use “ git init” command to initialize a git repo in the root of the folder
  • use “git add Readme.md ” to add it to the folder

Then commit your work

A commit is a record of what changes you have made or since the last time you made a commit.

  • git commit -m “add Readme.md”

Create a new branch

Branches allow you to move back and forth between ‘states’ of a project.

  • git branch -M main

Push a branch to GitHub

Now we will push the commit in your branch to GitHub repo. This will allow other people to see the changes you have made. To push changes to GitHub, we use

  • git remote add origin “add the link to your GitHub account”
  • Then use ‘git push -U origin main’ to push your work to GitHub

To see changes on git that your have made, we use git status
